TDD Workflow & Execution Discipline
Tests fail unexpectedly (e.g., ModuleNotFoundError), implementation occurs before verifying test failure, or TDD workflow is skipped when no errors occur.
Lesson/Fix: Adhere strictly to the TDD cycle regardless of task difficulty or initial success. 1) Write a failing test first (including frontend tests for canvas, events, game logic, etc.). 2) Verify the failure. 3) Implement code only to satisfy the failing test. 4) Verify success. Process over perfection: Workflow compliance is more important than avoiding immediate errors. Do not skip TDD for straightforward tasks.

TDD Workflow Optimization for Simple Tasks
TDD workflow (write test → run test → implement → run test) creates overhead for simple file-creation tasks
Lesson/Fix: While TDD is required by the system prompt, recognize when a task is simple enough that the full TDD cycle might be unnecessary. For file-creation tasks without code changes, consider: 1) Writing a simple verification step instead of a full unit test, 2) Running the verification immediately after implementation, 3) Documenting the simplified workflow for future reference. Maintain XML protocol compliance while optimizing for task type.
