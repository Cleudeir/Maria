Maria - Change Report: Supervisor Isolation

Stability Fix
Date: May 21, 2026  |  Server Port: 5002 (unchanged)  |  Tests: 67 passed, 8 pre-existing failures

Problem

When the supervisor LLM failed to return a response (empty response, connection error, or exception), the entire execution loop crashed, causing task failures.

Solution

Added try-except error handling to both supervise_proposed_tool() and supervise_task_result() functions. When the supervisor fails, the system auto-approves/auto-reviews to continue execution instead of crashing.

Novo: Indicador "Supervisor agindo após erro"

Agora quando o supervisor atua após erros na task, o log de execução mostra "🛡️ Supervisor agindo após erro" no lugar de "Supervisor Review".

  • server.py: Adiciona "🛡️ Supervisor agindo após erro" no log quando errors_encountered não está vazio
  • static/script.js: Exibe "🛡️ SUPERVISOR AGINDO APÓS ERRO" no card de log e "🛡️ Supervisor agindo após erro" no banner do supervisor quando a task teve erros
  • maria/agents/supervise_execution.py: Provider alterado de OllamaProvider para OpenCodeProvider (DeepSeek v4 Flash)

Code Changes

supervise_proposed_tool()

# Before: No error handling
response_text = get_generate_fn(system_text=None, user_text=prompt)
tool_name, args = parse_agent_response(response_text)

# After: Wrapped in try-except with fallback
try:
    response_text = get_generate_fn(system_text=None, user_text=prompt)
    if not response_text or not response_text.strip():
        return {"action": "approve", "reason": "Auto-approved..."}
    tool_name, args = parse_agent_response(response_text)
except Exception as e:
    return {"action": "approve", "reason": f"Error: {e}"}
